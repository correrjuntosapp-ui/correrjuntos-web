Por qué este diseño (en términos de ROI)

  • Hero adaptativo verde para usuario nuevo — responde "¿qué hago ahora?" sin que el usuario tenga que decidir entre 4 CTAs. Verde porque: gratis, "go", diferencia visual del naranja (que queda libre para quedadas/premium).
  • "Empieza a Moverte" como entry point único — es plan gratis, 6 semanas, mínima fricción. La data manda: 1% de tus users hace una run, 0,15% una quedada. La rampa empieza por planes, no por quedadas.
  • Stats "0/0/0" eliminados para nuevo — dato negativo en primer pliegue = mensaje "esta app no es para mí". Los stats vuelven cuando hay runs > 0.
  • Feed condicional — si follows = 0 ocultar la sección entera (no enseñar empty state). Para usuario activado vuelve y ocupa pantalla.
  • Quedadas con fallback de radio — Punta Umbría → Andalucía → España. Cap subido de 2 a 4. Badge "120 km" honesto.
  • PRO card blurred eliminado del primer pliegue — paywall prematuro al usuario que aún no ha registrado nada = friction sin razón. Vuelve cuando hay datos que desbloquear.

Cambios de código en HomeScreen.tsx

  • Nuevo componente HeroAdaptativo condicional por estado: userPlan ? sessionToday : noRuns ? newUserHero : welcomeBack
  • Stats card movido a componente StatsMini + render condicional {runs.length > 0 && ...}
  • combined.slice(0, 2) → slice(0, 4) + fallback de radio en getRecommendedQuedadas
  • Feed section: render solo si followingCount > 0 || feed.length > 0
  • Plans card → verde #10B981, sin icono, copy "Empieza a Moverte · 6 semanas · Gratis" cuando no hay plan activo
