# 1. Qué es CorrerJuntos

CorrerJuntos es una aplicación móvil de comunidad running que conecta corredores para salir a correr juntos. Disponible gratis en iOS y Android con modelo freemium (4,99€/mes premium).

No es solo un tracker GPS como Strava ni solo una red social. Es el puente entre querer correr y tener a alguien con quien hacerlo.

## El problema

El 70% de la gente que empieza a correr lo deja antes de 3 meses. La razón principal: correr solo es aburrido, duro y fácil de abandonar. Las apps actuales (Strava, Nike Run Club, Garmin Connect) están diseñadas para corredores individuales. Te dan datos pero no te dan a nadie con quien correr.

## La solución

- Quedadas de running geolocalizadas por ciudad, nivel y distancia
- Runner Matching: algoritmo que encuentra corredores compatibles
- 7 planes de entrenamiento con 1.025 workouts estructurados
- GPS tracking con audio alertas en segundo plano
- Sincronización con Garmin, COROS y Apple Watch
- Feed social: compartir actividades, likes, comentarios

# 2. Misión, Visión y Valores

## Misión

**Que nadie tenga que correr solo si no quiere.** Conectar a cada corredor con personas de su nivel, ritmo y zona para que correr sea una actividad social, motivadora y sostenible en el tiempo.

## Visión

Ser la app de referencia de la comunidad running en español. El lugar donde cualquier persona — desde un sedentario que quiere dar su primer paso hasta un maratoniano — encuentra su grupo.

## Valores

1. **Comunidad primero** — Cada funcionalidad se evalúa por si mejora la conexión entre corredores
2. **Accesibilidad** — Lo esencial es gratis. Premium mejora la experiencia, nunca la bloquea
3. **Inclusividad** — Todos los niveles, todas las edades. De caminar a maratón
4. **Datos reales** — GPS real, métricas reales, personas reales con fotos verificadas
5. **Simplicidad** — Abrir la app, encontrar una quedada, salir a correr

# 3. Cliente Ideal

## Perfil Primario — El Corredor Solitario (70% de usuarios)

- 25-55 años
- Corre 2-3 veces/semana pero siempre solo
- Tiene Strava pero no interactúa con nadie
- Quiere compañía pero no sabe cómo encontrarla
- No pertenece a ningún club de running
- Vive en ciudad mediana-grande en España o Latinoamérica

## Perfil Secundario — El Principiante Motivado (20%)

- 30-60 años
- Quiere empezar a correr pero no sabe cómo
- Le da vergüenza correr solo/a
- Busca un plan paso a paso + apoyo social
- Descarga la app por un artículo del blog

## Perfil Terciario — El Organizador de Grupo (10%)

- Corredor experimentado que ya organiza quedadas informales
- Quiere una herramienta para gestionar su grupo
- Crea quedadas, invita gente, registra actividades del grupo

## Unit Economics

- **CAC (coste adquisición):** ~0€ (SEO orgánico, sin ads)
- **Conversión free → premium:** 5-10% objetivo
- **LTV (6 meses):** 30€
- **Margen:** >90%

## Embudo de conversión

1. SEO: 400+ artículos generan 3.300 impresiones/día en Google
2. Landing pages de planes convierten visita en descarga
3. Plan 0→5K gratis engancha al usuario durante 8 semanas
4. Comunidad + quedadas retienen al usuario a largo plazo
5. Paywall premium al querer plan 10K+ o sync con reloj

# 6. Equipo: 10 Personas con IA

CorrerJuntos opera con un equipo de 10 personas potenciado por inteligencia artificial. Cada miembro utiliza herramientas de IA específicas para multiplicar su productividad x5-x10, permitiendo competir con equipos de 40-50 personas.

## Organigrama

```
              CEO / FOUNDER
          ┌────────┼────────┐
      PRODUCTO   CRECIMIENTO   NEGOCIO
      (3 pers)   (4 pers)     (2 pers)
                               + 1 Transversal
```

## ÁREA 1: PRODUCTO (3 personas)

### PERSONA 1: CEO / Founder / CTO

**Función:** Lidera el producto, toma decisiones de arquitectura, desarrolla features críticas. Define roadmap, prioriza backlog. Programa app, backend y web.

**Responsabilidades diarias:**
- Revisión de PRs y calidad de código
- Desarrollo de features críticas (planes, GPS, pagos)
- Decisiones de arquitectura y stack técnico
- Reuniones semanales con cada área
- Definir prioridades del sprint

**Herramientas IA:** Claude Code (desarrollo + automatización), GitHub Copilot (código), Cursor (IDE IA)

---

### PERSONA 2: UX/UI Designer

**Función:** Diseña todas las pantallas de la app, flujos de usuario, prototipos interactivos. Realiza tests de usabilidad con usuarios reales. Mantiene el design system (colores, tipografía, componentes).

**Responsabilidades diarias:**
- Diseñar nuevas pantallas y flujos en Figma
- Crear prototipos interactivos para validar con usuarios
- Mantener y actualizar el design system
- Colaborar con desarrollo para implementación pixel-perfect
- Realizar 2 tests de usabilidad por semana

**Herramientas IA:** Figma + plugins IA, Midjourney (conceptos visuales), Maze (user testing automatizado)

---

### PERSONA 3: QA Engineer

**Función:** Prueba cada release antes de subir a tiendas. Testing manual en iOS + Android + web. Escribe tests automatizados E2E. Verifica flows críticos.

**Responsabilidades diarias:**
- Testing manual de nuevas features en ambas plataformas
- Mantenimiento de la suite de tests automatizados
- Verificar: registro, GPS tracking, compra premium, planes, compartir
- Reportar bugs con pasos de reproducción exactos
- Validar fixes antes de cada release

**Herramientas IA:** Playwright + Claude (generación tests), BrowserStack (dispositivos), Detox (mobile E2E)
