✓ Por qué V4 sube del 7 al 8-9

  • Humanos visibles: avatars reales de Marta, Carlos, Juanan. La app se siente social
  • Mapas de recorridos: polylines ember con gradient + marker inicio/fin. El corazón visual de Strava
  • Compatibilidad explícita: chip "94% COMPAT" · mono ember. El algoritmo que ya tienes, visible
  • Contexto de relación: chip COMPATIBLE / COINCIDISTEIS / SIGUES. El user entiende por qué le muestra a cada runner
  • Stats estilo Strava: KM / Tiempo / Min-km con divisores hairline
  • Engagement visible: kudos count + comentarios + "Seguir" → invita a interactuar
  • Privacy respected: el mapa de Carlos está blurred con "MAPA PRIVADO" (usa el map_visible que ya calcula el RPC)
  • Distancia geográfica: "4 km de ti" vs "612 km de ti" · el user sabe si es alcanzable
  • Planes relegado a card compacta (ya no protagonista). Las actividades de la gente son el show

Lo que tu backend YA soporta

  • get_feed_with_compatible(p_limit, p_offset) RPC → devuelve runs de followed + known + compatible
  • polyline_encoded en tabla runs → ruta GPS comprimida
  • compat_score calculado por compatibility_score()
  • relation: 'follow' / 'known' / 'compatible'
  • map_visible calculado server-side desde privacy_settings
  • distance_km_from_me → distancia en km del user al runner que creó la activity

La realidad · qué hace falta para que este feed esté lleno

  • Users que corran regularmente → mínimo 5-10 activos al mes para que el feed tenga contenido
  • Users con privacy_settings.activity_visibility = 'public' o 'followers' y tú siguiendo
  • Polylines reales grabadas → ocurre cuando registran carrera con GPS (ya funciona)
  • Sin users reales activos = feed vacío = mismo problema que antes. Aplicar este V4 SIN la quedada física + email reactivation = el feed estará vacío y volverás a estar en 4/10 con empty state
  • Con este V4 aplicado DESPUÉS de 10 users activos en Huelva = 8-9/10 real
Conclusión experta:
V4 es lo correcto para el diseño. Pero sin usuarios reales corriendo, será V4 con feed vacío.
Orden óptimo: primero comunidad (quedada + email reactivation), luego aplicar V4.
