Por qué Lucide > emoji en apps premium

  • ●Color controlable: puedes teñir el icono ember/paper/ash sin que cambie la forma. Emoji = color fijo del OS
  • ●Stroke width customizable: 1.5 / 1.8 / 2 px según peso visual necesario. Emoji = peso fijo
  • ●Consistencia iOS + Android: mismo pixel-perfect render. Emoji cambia según sistema (iOS 🏃 ≠ Android 🏃 ≠ WhatsApp 🏃)
  • ●Animable: puedes animar stroke-dasharray, rotation, morph. Emoji = estático
  • ●Tamaño escalable: 12px a 64px sin pixelación. Emoji pierde calidad en tamaños pequeños
  • ●Apps pro usan esto: Strava, Linear, Vercel, Arc, Notion — todos usan icon systems (Lucide/Phosphor/Custom SVG), ninguno usa emojis en UI

Implementación técnica

Buena noticia: ya usáis SVG inline en varias screens del onboarding (Hook, Profile, Photo). Solo falta:

  1. npm install lucide-react-native
  2. Reemplazar emojis en Home/Feed/Map/Profile por imports tipo import { Home, Bell, Activity } from 'lucide-react-native'
  3. Usarlos con props unificados: <Bell size={20} color="#f97316" strokeWidth={1.7} />

Paquete ligero (~30KB tree-shakeable). Compatible 100% con React Native + Expo.
